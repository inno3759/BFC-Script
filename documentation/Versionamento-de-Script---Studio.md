O versionamento de um script nada mais é do que a visualização das versões realizadas no artefato, essa informação fica localizada na tela que contém as informações do script. 

![versionamento](/assets/images/script54-9dd08274892dbc9f3214548796b8e8e5.png)  
---  
  
Cada vez que algo é alterado no artefato e publicado será criado uma nova versão, e a versão atual será executada ao clicar no botão **EXECUTAR**.

![versionamento](/assets/images/script55-aedf8ef47aca0a69ce1a1991bd9648a1.png)  
---  
  
Todo o histórico de versões realizados nas publicações são apresentadas no centro da tela na aba **ATUALIZAÇÕES** , cada alteração realizada no script a ferramenta mantém o histórico neste ambiente. As informações são exibidas através de colunas facilitando a visualização e a consulta realizada pelo usuário.

Pressionando o ícone simbolizado por um olho é mostrado exatamente como estava escrito em cada uma das versões publicadas no script. 

![versionamento](/assets/images/script56-1478f1c97b3ab37bd79069ff7766d083.png)  
---  
  
IMPORTANTE!

Na ferramenta de Scripts não é possível restaurar uma versão, só é possível visualizá-la. Para voltar uma versão anterior é necessário copiar todo o código de uma versão desejada e colar em cima da versão atual e publicar, assim uma nova versão será gerada. O versionamento é igual para Scripts, Componentes, Críticas e Fontes Dinâmicas.

Quando executado um relatório do tipo **Fonte dinâmica** , será sempre a versão atual do script, ou seja, se o script for atualizado e publicado será essa versão que será apresentada no relatório.

Uma funcionalidade que o usuário deve se atentar é relacionado aos parâmetros já cadastrados. Caso seja editado um ou mais parâmetros em uma determinada versão é necessário reescrevê-los manualmente na versão atual.
